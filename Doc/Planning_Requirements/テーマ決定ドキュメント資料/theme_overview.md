# テーマ概要

## 商品在庫管理システム

### 概要
商品在庫管理システムは、ユーザーが在庫を簡単に管理・確認できるようにするためWEBシステムです。
このシステムでは、在庫状況の可視化、効率的な管理、そして正確なデータ分析を可能にします。

### 主な特徴
- **在庫状況の可視化**: 商品の在庫数や状態をリアルタイムで確認可能。
- **効率的な管理**: 商品の追加、削除、更新を簡単に行えるユーザインターフェースを提供。
- **データ分析**: 売上や在庫回転率などのデータを分析し、ビジネスの意思決定をサポート。（第２フェーズ）
- **CSVインポート・エクスポート**: 商品データを一括でインポート・エクスポート可能にし、データ管理を効率化。

### 目的
このシステムの目的は、在庫管理における手間を削減し、正確性を向上させることで、ユーザーの業務効率を最大化することです。

### 差別化ポイント
1.ログイン／ユーザー管理
・Spring Securityの活用: 実務レベルのセキュリティを実現し、認証・認可の理解をアピール。
・権限管理の柔軟性: 管理者と一般ユーザーの役割を明確化し、実務に即した管理機能を提供。（第２フェーズ）
・セキュリティ意識の高さ: パスワードのハッシュ化や多要素認証の導入を検討。

2.商品マスタ管理
・論理削除の採用: 商品データの完全削除を防ぎ、データの追跡可能性を確保。
・カテゴリ管理の柔軟性: カテゴリの階層構造やフィルタリング機能を追加。
・ステータス管理: 商品の取扱状況をリアルタイムで反映し、業務効率を向上。

3.在庫管理（コア機能）★★★ ⇒ 今回のシステム利用のメインのため、3つ星
・在庫不足アラート: 在庫数が一定以下になった際にアラートを表示し、業務の迅速な対応を支援。
・入出庫履歴の管理: stock_transactions テーブルを活用し、在庫の変動履歴を可視化。
・自動計算機能: 入庫・出庫処理時に在庫数を自動更新し、人的ミスを削減。
・リアルタイム在庫表示: 商品ごとの在庫状況を即座に確認可能にするダッシュボードを提供。

### 機能開発の優先度
在庫管理：高、商品マスタ管理：中、ログイン／ユーザー管理：低

### 差別化の具体的なアプローチ
ユーザー体験の向上: シンプルで直感的なUI/UX設計を採用。
データ分析機能の追加: 在庫回転率や売上分析を行えるレポート機能を提供。（第２フェーズ）
拡張性の確保: 将来的な機能追加を見据えた柔軟な設計。

### ユーザーの課題やニーズ

#### ユーザーの課題(一般的な課題)
- **在庫状況の把握が困難**:
  - リアルタイムで在庫数を確認できない。
  - 在庫不足や過剰在庫に気づくのが遅れる。
- **入出庫管理の煩雑さ**:
  - 手動での在庫記録に時間がかかり、ミスが発生しやすい。
  - 入出庫履歴を追跡するのが難しい。
- **データ分析の不足**:
  - 在庫回転率や売上データを分析する仕組みがない。
  - 過去のデータを基にした在庫予測ができない。
- **操作性の問題**:
  - システムが複雑で使いにくい。
  - 必要な情報にすぐアクセスできない。

#### ユーザーのニーズ
- **リアルタイム在庫表示**:
  - 商品ごとの在庫状況を即座に確認できるダッシュボード。
- **在庫不足アラート**:
  - 在庫数が一定以下になった際に通知を受け取る機能。
- **効率的な入出庫管理**:
  - 入庫・出庫処理を簡単に行えるインターフェース。
  - 入出庫履歴を自動で記録し、追跡可能にする。
- **データ分析機能**:
  - 在庫回転率や売上データを可視化するレポート機能。
  - 過去データを基にした在庫予測機能。
- **操作性の向上**:
  - シンプルで直感的なUI/UX。
  - 必要な情報にすぐアクセスできる検索・フィルタリング機能。

#### 課題とニーズの対応表
| **課題**                     | **対応する機能**                          |
|-----------------------------|---------------------------------------|
| 在庫状況の把握が困難          | リアルタイム在庫表示、在庫不足アラート            |
| 入出庫管理の煩雑さ            | 入出庫管理機能、履歴管理、自動計算機能            |
| データ分析の不足              | 在庫回転率・売上分析、レポート生成、在庫予測機能   |
| 操作性の問題                  | シンプルなUI/UX、検索・フィルタリング機能         |

#### 競合アプリのリサーチと成功基準の定義

##### 競合アプリのリサーチ

#####　リサーチした競合アプリ（5社）
1. **ZAICO（ザイコ）**: 日本の中小企業向け在庫管理アプリ。低コストで使いやすいが、データ分析機能が限定的。
2. **弥生販売（実務フロー比較）**: 日本の中小企業向け販売管理ソフト。会計連携が強みだが、デスクトップ版が中心でクラウド対応が遅れている。
3. **ロジクラ**: EC事業者向けの物流・在庫管理システム。外部連携が豊富だが、料金が高め。
4. **Zoho Inventory**: グローバル展開の総合在庫管理システム。機能充実だが、日本語対応が不完全で複雑。
5. **スマートマットクラウド**: IoTマットで自動計測するシステム。省人化に貢献するが、初期投資が高く用途が限定的。

###### 1. ZAICO（ザイコ）
- **概要**: 日本発の在庫管理アプリ。中小企業向けに特化したクラウド型サービス。
- **主な機能**:
  - バーコード・QRコードを使った在庫管理
  - リアルタイム在庫数の確認
  - CSV一括登録・エクスポート
  - 複数拠点管理
  - 在庫アラート機能
- **料金**: 無料プラン〜月額3,980円（プランにより機能制限あり）
- **強み**: 
  - 日本語UIで使いやすい
  - スマホアプリで現場での操作が容易
  - 導入コストが低い
- **弱み**: 
  - データ分析機能が限定的
  - カスタマイズ性が低い
  - 入出庫履歴の詳細分析が不足

###### 2. 弥生販売（実務フロー比較）
- **概要**: 弥生株式会社が提供する販売管理ソフト。見積・受発注・在庫管理・請求まで一元管理。
- **主な機能**:
  - 見積・受発注管理
  - 在庫管理・入出庫管理
  - 請求書・納品書発行
  - 弥生会計との連携
  - 実務に即した業務フロー
- **料金**: 買い切り型40,000円〜、クラウド版月額2,600円〜
- **強み**: 
  - 日本の商習慣に完全対応
  - 会計ソフトとのシームレスな連携
  - 実務フローに沿った設計
  - サポート体制が充実
- **弱み**: 
  - デスクトップ版が主流でクラウド対応が遅れている
  - UIが古く、操作性に課題
  - リアルタイム性に欠ける
  - モバイル対応が不十分

###### 3. ロジクラ
- **概要**: 物流・在庫管理に特化したクラウドシステム。EC事業者向けに強い。
- **主な機能**:
  - 在庫管理・入出荷管理
  - 送り状発行
  - ピッキングリスト作成
  - 複数倉庫管理
  - API連携（楽天、Amazon等）
- **料金**: 月額9,900円〜（従量課金制）
- **強み**: 
  - EC事業者向けの機能が充実
  - 外部サービスとの連携が豊富
  - 物流業務全般をカバー
- **弱み**: 
  - EC事業者以外には機能が過剰
  - 料金が高め
  - 小規模事業者には導入ハードルが高い

###### 4. Zoho Inventory（国際的競合）
- **概要**: グローバルに展開する総合在庫管理システム。
- **主な機能**:
  - 在庫管理・注文管理
  - 複数倉庫・複数通貨対応
  - 販売・購買管理
  - レポート・分析機能
  - 会計ソフト連携
- **料金**: 月額1,500円〜（プランにより異なる）
- **強み**: 
  - 機能が非常に充実
  - グローバル対応
  - 拡張性が高い
- **弱み**: 
  - 日本語対応が不完全
  - 機能が多すぎて使いこなすのが難しい
  - 初期設定に時間がかかる

###### 5. スマートマットクラウド
- **概要**: IoTマットで在庫を自動計測するシステム。
- **主な機能**:
  - 重量センサーによる自動在庫計測
  - リアルタイム在庫把握
  - 自動発注機能
  - アラート通知
- **料金**: 月額5,000円〜（マット購入が必要）
- **強み**: 
  - 自動計測で手間が少ない
  - IoT技術の活用
  - 省人化に貢献
- **弱み**: 
  - 初期投資が高い（マット購入）
  - 重量で測定できる商品に限定
  - 細かい在庫管理には不向き
